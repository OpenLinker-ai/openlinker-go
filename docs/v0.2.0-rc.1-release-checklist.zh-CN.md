## 2. Core 契约核对

- [x] Core `main` 与 SDK 使用同一 `/api/v1/agent-runtime/*` 路由。
- [x] handshake contract ID、protocol version 和 digest 一致。
- [x] Agent registration 的 `connection_mode=runtime` 与 Runtime transport 选择互相独立；旧 `runtime_ws/runtime_pull/agent_node` 只在 SDK 注册兼容层归一化。
- [x] assignment、ack、lease、event、result、resume、cancel 和 drain payload 字段一致。
- [x] 认证、mTLS、contract 错误不会被 SDK 分类为可 fallback 的传输错误。

核对证据见 [runtime-core-compatibility.zh-CN.md](./runtime-core-compatibility.zh-CN.md)。

- [x] `go test ./...` 与 `go vet ./...` 通过。

## 4. RC 发布操作

执行前必须获得仓库维护者明确确认：

```bash
git status --short
git tag -a v0.2.0-rc.1 -m "openlinker-go v0.2.0-rc.1"
git push origin HEAD
git push origin v0.2.0-rc.1
```

- [ ] tag 指向已通过全部检查的提交。
- [ ] Go proxy 能解析 `github.com/OpenLinker-ai/openlinker-go@v0.2.0-rc.1`。
- [ ] 在干净目录执行 `go get ...@v0.2.0-rc.1` 和最小编译验证。
- [ ] layout 删除临时 replace 后重新执行全量测试。
